  <action>
Create three YAML manifests in k8s/base/:

**1. storage-class-immediate.yaml** -- Custom StorageClass for immediate volume binding:
```yaml
apiVersion: storage.k8s.io/v1
kind: StorageClass
metadata:
  name: standard-immediate
  labels:
    managed-by: argocd
provisioner: pd.csi.storage.gke.io
parameters:
  type: pd-standard
  fstype: ext4
volumeBindingMode: Immediate
reclaimPolicy: Retain
allowVolumeExpansion: true
allowedTopologies:
  - matchLabelExpressions:
      - key: topology.gke.io/zone
        values:
          - us-central1-a
          - us-central1-b
          - us-central1-c
```

IMPORTANT: The zone values (us-central1-a, us-central1-b, us-central1-c) are placeholders. Before creating this file, query the actual cluster node zones:
```bash
kubectl get nodes -o custom-columns=NAME:.metadata.name,ZONE:.metadata.labels.topology\.gke\.io/zone --no-headers 2>/dev/null
```
If kubectl is not available or returns an error, use the placeholder zones above and add a YAML comment noting they must be verified: `# TODO: Verify zones match client-cluster nodes (kubectl get nodes -L topology.gke.io/zone)`

Key requirements met:
- STOR-03: volumeBindingMode: Immediate (NOT WaitForFirstConsumer)
- STOR-04: reclaimPolicy: Retain (prevents data loss on PVC deletion)
- STOR-05: allowedTopologies restricts to node zones

**2. pvc-postgres.yaml** -- Postgres 10Gi PVC:
```yaml
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: postgres-data
  namespace: firecrawl
  labels:
    app: postgres
    component: database
    managed-by: argocd
spec:
  accessModes:
    - ReadWriteOnce
  storageClassName: standard-immediate
  resources:
    requests:
      storage: 10Gi
```

Key requirements met:
- STOR-01: 10Gi storage for Postgres
- storageClassName: standard-immediate links to custom StorageClass

**3. pvc-redis.yaml** -- Redis 1Gi PVC:
```yaml
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: redis-data
  namespace: firecrawl
  labels:
    app: redis
    component: cache
    managed-by: argocd
spec:
  accessModes:
    - ReadWriteOnce
  storageClassName: standard-immediate
  resources:
    requests:
      storage: 1Gi
```

Key requirements met:
- STOR-02: 1Gi storage for Redis
- storageClassName: standard-immediate links to custom StorageClass

Do NOT use ReadWriteMany -- GKE pd-standard only supports ReadWriteOnce.
Do NOT use the default storage class -- it uses WaitForFirstConsumer which violates STOR-03.
Do NOT set reclaimPolicy to Delete -- would cause data loss (violates STOR-04).
