Replace the existing stub api-deployment.yaml with the full Deployment manifest. The file must contain exactly these specifications:

**Metadata:**
- name: firecrawl-api
- namespace: firecrawl
- labels: app: firecrawl-api, component: api, version: v1

**Spec:**
- replicas: 1
- strategy: RollingUpdate with maxUnavailable: 0, maxSurge: 1
- selector matchLabels: app: firecrawl-api

**Pod template labels:** app: firecrawl-api, component: api, version: v1

**serviceAccountName:** firecrawl-api

**Init containers (two, in this order):**

1. wait-for-postgres:
   - image: busybox:latest
   - command: sh -c with until loop: `until nc -z postgres-service.firecrawl.svc.cluster.local 5432; do echo "Waiting for postgres..."; sleep 2; done; echo "Postgres is ready"`

2. wait-for-redis:
   - image: busybox:latest
   - command: sh -c with until loop: `until nc -z redis-service.firecrawl.svc.cluster.local 6379; do echo "Waiting for redis..."; sleep 2; done; echo "Redis is ready"`

**Main container:**
- name: firecrawl-api
- image: firecrawl-api (bare name; Kustomize image transformer will rewrite to GCR path with SHA)
- imagePullPolicy: IfNotPresent
- ports: containerPort 3002, name http, protocol TCP
- env:
  - NODE_OPTIONS: "--max-old-space-size=6800" (85% of 8Gi limit = 6963Mi, using 6800Mi for safety)
- envFrom (this exact list, in this order):
  - configMapRef name: firecrawl-database
  - configMapRef name: firecrawl-redis
  - configMapRef name: firecrawl-application
  - secretRef name: firecrawl-database-secret
  - secretRef name: firecrawl-api-secrets
- resources:
  - requests: cpu 1, memory 2Gi
  - limits: cpu 4, memory 8Gi
- readinessProbe:
  - httpGet path: /liveness, port: 3002
  - initialDelaySeconds: 30, periodSeconds: 10, timeoutSeconds: 5, failureThreshold: 3
- livenessProbe:
  - httpGet path: /liveness, port: 3002
  - initialDelaySeconds: 60, periodSeconds: 20, timeoutSeconds: 5, failureThreshold: 3

IMPORTANT: Do NOT reference a secret called "firecrawl-redis-secret" or "firecrawl-api-secret" (singular). The correct names are "firecrawl-database-secret" and "firecrawl-api-secrets" (plural) as documented in k8s/base/secrets-README.md.
