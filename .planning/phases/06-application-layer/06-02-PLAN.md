Output: extract-worker-deployment.yaml, nuq-worker-deployment.yaml, prefetch-worker-deployment.yaml

Create k8s/base/extract-worker-deployment.yaml with these exact specifications:

**Metadata:**
- name: extract-worker
- namespace: firecrawl
- labels: app: extract-worker, component: worker, worker-type: extract

**Spec:**
- replicas: 1
- selector matchLabels: app: extract-worker

**Pod template labels:** app: extract-worker, component: worker, worker-type: extract

**serviceAccountName:** firecrawl-worker

**Init containers (same pattern as API):**
1. wait-for-postgres: busybox:latest, `until nc -z postgres-service.firecrawl.svc.cluster.local 5432; do echo "Waiting for postgres..."; sleep 2; done; echo "Postgres is ready"`
2. wait-for-redis: busybox:latest, `until nc -z redis-service.firecrawl.svc.cluster.local 6379; do echo "Waiting for redis..."; sleep 2; done; echo "Redis is ready"`

**Main container:**
- name: extract-worker
- image: firecrawl-api (same image as API, Kustomize transforms it)
- imagePullPolicy: IfNotPresent
- command: ["node"]
- args: ["dist/src/services/extract-worker.js"]
- ports: containerPort 3004, name http, protocol TCP
- env:
  - NODE_OPTIONS: "--max-old-space-size=6800"
  - EXTRACT_WORKER_PORT: "3004"
- envFrom (same as API):
  - configMapRef name: firecrawl-database
  - configMapRef name: firecrawl-redis
  - configMapRef name: firecrawl-application
  - secretRef name: firecrawl-database-secret
  - secretRef name: firecrawl-api-secrets
- resources: requests cpu 1, memory 4Gi; limits cpu 2, memory 8Gi
- readinessProbe: httpGet path /health, port 3004, initialDelaySeconds 30, periodSeconds 10, timeoutSeconds 5, failureThreshold 3
- livenessProbe: httpGet path /health, port 3004, initialDelaySeconds 60, periodSeconds 20, timeoutSeconds 5, failureThreshold 3
