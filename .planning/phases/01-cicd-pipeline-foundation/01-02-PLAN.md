Create `.github/workflows/ci-build-deploy.yml` with the following exact structure.

**Top-level configuration:**
```yaml
name: CI Build and Deploy

on:
  push:
    branches:
      - main
    paths:
      - 'apps/api/**'
      - 'apps/ui/ingestion-ui/**'
      - '.github/workflows/ci-build-deploy.yml'
  workflow_dispatch:

concurrency:
  group: deploy-${{ github.ref }}
  cancel-in-progress: true

env:
  GCP_PROJECT_ID: prometheus-461323
  GCR_REGISTRY: gcr.io
```

- Trigger: push to main on relevant paths (NOT k8s/** -- that would re-trigger on manifest commits)
- workflow_dispatch: allows manual trigger for testing
- concurrency: cancel in-progress runs when new push arrives (prevents parallel manifest conflicts)

**Job: build-push-update**
```yaml
jobs:
  build-push-update:
    runs-on: ubuntu-latest
    permissions:
      contents: write
      id-token: write
```

- `contents: write` -- needed to push manifest commits back to repo
- `id-token: write` -- needed for Workload Identity Federation OIDC token exchange

**Matrix strategy for parallel builds:**
```yaml
    strategy:
      matrix:
        service:
          - name: firecrawl-api
            context: ./apps/api
            dockerfile: ./apps/api/Dockerfile
          - name: ingestion-ui
            context: ./apps/ui/ingestion-ui
            dockerfile: ./apps/ui/ingestion-ui/Dockerfile
```

**Steps (in exact order):**

1. **Checkout** with full history for git operations:
```yaml
      - name: Checkout repository
        uses: actions/checkout@v4
        with:
          fetch-depth: 0
```

2. **Extract short SHA** (7 characters for immutable tag):
```yaml
      - name: Extract short SHA
        run: echo "SHORT_SHA=${GITHUB_SHA:0:7}" >> $GITHUB_ENV
```

3. **Authenticate to GCP via Workload Identity Federation:**
```yaml
      - name: Authenticate to Google Cloud
        uses: google-github-actions/auth@v2
        with:
          workload_identity_provider: ${{ secrets.WIF_PROVIDER }}
          service_account: ${{ secrets.WIF_SERVICE_ACCOUNT }}
```

4. **Setup gcloud CLI:**
```yaml
      - name: Set up Cloud SDK
        uses: google-github-actions/setup-gcloud@v2
```

5. **Configure Docker for GCR:**
```yaml
      - name: Configure Docker for GCR
        run: gcloud auth configure-docker gcr.io
```

6. **Setup Docker Buildx:**
```yaml
      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3
```

7. **Build and push image:**
```yaml
      - name: Build and push image
        uses: docker/build-push-action@v6
        with:
          context: ${{ matrix.service.context }}
          file: ${{ matrix.service.dockerfile }}
          push: true
          tags: ${{ env.GCR_REGISTRY }}/${{ env.GCP_PROJECT_ID }}/${{ matrix.service.name }}:${{ env.SHORT_SHA }}
          cache-from: type=gha
          cache-to: type=gha,mode=max
```

Single platform (linux/amd64) -- GKE uses amd64 nodes. No multi-arch needed.

8. **Verify image availability in GCR with retry:**
```yaml
      - name: Verify image in GCR
        run: |
          image="${GCR_REGISTRY}/${GCP_PROJECT_ID}/${{ matrix.service.name }}:${SHORT_SHA}"
          for i in 1 2 3 4 5; do
            echo "Verification attempt $i of 5"
            if gcloud container images describe "$image" --format='get(image_summary.digest)'; then
              echo "Image verified: $image"
              exit 0
            fi
            echo "Image not yet available, waiting 10 seconds..."
            sleep 10
          done
          echo "Failed to verify image after 5 attempts"
          exit 1
```

9. **Install kustomize:**
```yaml
      - name: Install kustomize
        run: |
          curl -s "https://raw.githubusercontent.com/kubernetes-sigs/kustomize/master/hack/install_kustomize.sh" | bash
          sudo mv kustomize /usr/local/bin/
          kustomize version
```

10. **Update kustomization.yaml with new image tag:**
```yaml
      - name: Update kustomization with new image
        run: |
          cd k8s/base
          kustomize edit set image \
            ${{ matrix.service.name }}=${GCR_REGISTRY}/${GCP_PROJECT_ID}/${{ matrix.service.name }}:${SHORT_SHA}
          echo "Updated kustomization.yaml:"
          cat kustomization.yaml
```

11. **Commit and push manifest changes with [skip ci]:**
```yaml
      - name: Commit and push manifest updates
        run: |
          git config user.name "github-actions[bot]"
          git config user.email "github-actions[bot]@users.noreply.github.com"

          git add k8s/base/kustomization.yaml

          if git diff --staged --quiet; then
            echo "No changes to commit"
            exit 0
          fi

          git commit -m "ci: update ${{ matrix.service.name }} image to ${SHORT_SHA} [skip ci]"

          for i in 1 2 3; do
            if git push origin main; then
              echo "Successfully pushed manifest update"
              exit 0
            fi
            echo "Push failed (attempt $i/3), pulling and retrying..."
            git pull --rebase origin main
          done

          echo "Failed to push after 3 attempts"
          exit 1
```

Important details:
- `git add k8s/base/kustomization.yaml` -- only stage the kustomization file, not everything
- `git diff --staged --quiet` -- skip commit if no changes (idempotent)
- `[skip ci]` in commit message -- prevents re-triggering this workflow
- `git pull --rebase` retry loop -- handles concurrent push from parallel matrix job

Human must configure GCP Workload Identity Federation and add GitHub repository secrets. The CI workflow references `secrets.WIF_PROVIDER` and `secrets.WIF_SERVICE_ACCOUNT` which must be set before the workflow can authenticate to GCP.

**GCP Setup (if WIF pool does not already exist):**

1. Get your GCP project number:
   ```bash
   gcloud projects describe prometheus-461323 --format='value(projectNumber)'
   ```

2. Create Workload Identity Pool:
   ```bash
   gcloud iam workload-identity-pools create github-actions-pool \
     --location=global \
     --display-name="GitHub Actions Pool"
   ```

3. Create OIDC Provider:
   ```bash
   gcloud iam workload-identity-pools providers create-oidc github-provider \
     --location=global \
     --workload-identity-pool=github-actions-pool \
     --issuer-uri=https://token.actions.githubusercontent.com \
     --attribute-mapping="google.subject=assertion.sub,attribute.repository=assertion.repository"
   ```

4. Create service account:
   ```bash
   gcloud iam service-accounts create github-actions \
     --display-name="GitHub Actions CI"
   ```

5. Grant GCR push permissions:
   ```bash
   gcloud projects add-iam-policy-binding prometheus-461323 \
     --member="serviceAccount:github-actions@prometheus-461323.iam.gserviceaccount.com" \
     --role="roles/storage.admin"
   ```

6. Bind service account to WIF pool (replace PROJECT_NUMBER and OWNER):
   ```bash
   gcloud iam service-accounts add-iam-policy-binding \
     github-actions@prometheus-461323.iam.gserviceaccount.com \
     --role="roles/iam.workloadIdentityUser" \
     --member="principalSet://iam.googleapis.com/projects/PROJECT_NUMBER/locations/global/workloadIdentityPools/github-actions-pool/attribute.repository/OWNER/firecrawl"
   ```

**GitHub Secrets Setup:**

7. Get WIF provider resource name:
   ```bash
   gcloud iam workload-identity-pools providers describe github-provider \
     --location=global \
     --workload-identity-pool=github-actions-pool \
     --format='value(name)'
   ```

8. Add two GitHub repository secrets (Settings -> Secrets and variables -> Actions):
   - `WIF_PROVIDER`: Full provider resource name (format: `projects/NUMBER/locations/global/workloadIdentityPools/github-actions-pool/providers/github-provider`)
   - `WIF_SERVICE_ACCOUNT`: `github-actions@prometheus-461323.iam.gserviceaccount.com`
