Create a multi-stage production Dockerfile at `apps/ui/ingestion-ui/Dockerfile` for the Vite React app.

**Stage 1: builder (node:22-slim)**
```dockerfile
FROM node:22-slim AS builder
ENV PNPM_HOME="/pnpm"
ENV PATH="$PNPM_HOME:$PATH"
RUN corepack enable
WORKDIR /app
```

Copy only what's needed for dependency install first (layer caching):
```dockerfile
COPY pnpm-lock.yaml package.json ./
RUN --mount=type=cache,id=pnpm,target=/pnpm/store pnpm install --frozen-lockfile
```

Then copy source and build:
```dockerfile
COPY . .
RUN pnpm run build
```

Note: The ingestion-ui is a standalone app (not part of a pnpm workspace in its build context), so copy its own pnpm-lock.yaml and package.json. The build command is `tsc -b && vite build` which outputs to `dist/`.

**Stage 2: production (nginx:alpine)**
```dockerfile
FROM nginx:alpine
COPY --from=builder /app/dist /usr/share/nginx/html
```

Add nginx config for SPA routing (try_files fallback to index.html):
```
server {
    listen 80;
    root /usr/share/nginx/html;
    index index.html;
    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

Write this config to `/etc/nginx/conf.d/default.conf` using a heredoc or RUN echo command.

```dockerfile
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

Important: The build context will be `./apps/ui/ingestion-ui` (same pattern as API), so all COPY paths are relative to that directory. The ingestion-ui directory has its own `pnpm-lock.yaml` -- verify this exists. If not, use the approach of copying the workspace root's lock file (adjust COPY accordingly and document).

Also add a `.dockerignore` at `apps/ui/ingestion-ui/.dockerignore` with:
```
node_modules
dist
.git
```

Create the `k8s/base/` directory and three files.

**File 1: k8s/base/kustomization.yaml**
```yaml
apiVersion: kustomize.config.k8s.io/v1beta1
kind: Kustomization

resources:
  - api-deployment.yaml
  - ui-deployment.yaml

images:
  - name: firecrawl-api
    newName: gcr.io/prometheus-461323/firecrawl-api
    newTag: "initial"
  - name: ingestion-ui
    newName: gcr.io/prometheus-461323/ingestion-ui
    newTag: "initial"
```

The `images` section is critical -- `kustomize edit set image firecrawl-api=gcr.io/prometheus-461323/firecrawl-api:NEWSHA` will update `newTag` for the matching `name` entry. The `name` field must match what's used in deployment manifests' `image:` field.

**File 2: k8s/base/api-deployment.yaml**
```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: firecrawl-api
  namespace: firecrawl
  labels:
    app: firecrawl-api
spec:
  replicas: 1
  selector:
    matchLabels:
      app: firecrawl-api
  template:
    metadata:
      labels:
        app: firecrawl-api
    spec:
      containers:
        - name: firecrawl-api
          image: firecrawl-api
          ports:
            - containerPort: 8080
```

Note: The `image: firecrawl-api` (without registry prefix) is intentional -- Kustomize will replace it with `gcr.io/prometheus-461323/firecrawl-api:SHA` at build time via the images transformer. Port 8080 matches the EXPOSE in apps/api/Dockerfile.

**File 3: k8s/base/ui-deployment.yaml**
```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: ingestion-ui
  namespace: firecrawl
  labels:
    app: ingestion-ui
spec:
  replicas: 1
  selector:
    matchLabels:
      app: ingestion-ui
  template:
    metadata:
      labels:
        app: ingestion-ui
    spec:
      containers:
        - name: ingestion-ui
          image: ingestion-ui
          ports:
            - containerPort: 80
```

Note: `image: ingestion-ui` matches the `name` in kustomization.yaml images section. Port 80 matches nginx in the ingestion-ui Dockerfile. These are stubs -- Phase 6 will add resource limits, probes, env vars, init containers.
