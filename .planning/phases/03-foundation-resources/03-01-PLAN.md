**File 1 - k8s/base/serviceaccounts.yaml:**

Create 4 ServiceAccount documents separated by `---`:

1. `firecrawl-api` - namespace: firecrawl, labels: app=firecrawl component=api, automountServiceAccountToken: true
2. `firecrawl-worker` - namespace: firecrawl, labels: app=firecrawl component=worker, automountServiceAccountToken: true
3. `firecrawl-ui` - namespace: firecrawl, labels: app=firecrawl component=ui, automountServiceAccountToken: false
4. `firecrawl-playwright` - namespace: firecrawl, labels: app=firecrawl component=playwright, automountServiceAccountToken: false

Each ServiceAccount uses apiVersion: v1, kind: ServiceAccount. API and worker get token mount (may need ConfigMap access). UI and playwright do not (no K8s API access needed).

**File 2 - k8s/base/rbac.yaml:**

Create 3 YAML documents:

Document 1 - Role for read-only ConfigMap access:
```yaml
apiVersion: rbac.authorization.k8s.io/v1
kind: Role
metadata:
  name: firecrawl-configmap-reader
  namespace: firecrawl
rules:
  - apiGroups: [""]
    resources: ["configmaps"]
    verbs: ["get", "list", "watch"]
```

Document 2 - RoleBinding for firecrawl-api:
```yaml
apiVersion: rbac.authorization.k8s.io/v1
kind: RoleBinding
metadata:
  name: firecrawl-api-configmap-binding
  namespace: firecrawl
subjects:
  - kind: ServiceAccount
    name: firecrawl-api
    namespace: firecrawl
roleRef:
  kind: Role
  name: firecrawl-configmap-reader
  apiGroup: rbac.authorization.k8s.io
```

Document 3 - RoleBinding for firecrawl-worker:
```yaml
apiVersion: rbac.authorization.k8s.io/v1
kind: RoleBinding
metadata:
  name: firecrawl-worker-configmap-binding
  namespace: firecrawl
subjects:
  - kind: ServiceAccount
    name: firecrawl-worker
    namespace: firecrawl
roleRef:
  kind: Role
  name: firecrawl-configmap-reader
  apiGroup: rbac.authorization.k8s.io
```

No Role/RoleBinding for firecrawl-ui or firecrawl-playwright (secure default: no permissions).

**File 3 - Update k8s/base/kustomization.yaml:**

Add namespace.yaml, serviceaccounts.yaml, and rbac.yaml to the resources list. Keep existing entries (api-deployment.yaml, ui-deployment.yaml) and the images section intact. The updated resources list should be:
```yaml
resources:
  - namespace.yaml
  - serviceaccounts.yaml
  - rbac.yaml
  - api-deployment.yaml
  - ui-deployment.yaml
```

Namespace resources listed first so they are created before deployments that reference them.
