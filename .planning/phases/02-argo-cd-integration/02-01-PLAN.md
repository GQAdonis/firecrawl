<objective>
Create the Argo CD Application manifest that configures GitOps continuous deployment for firecrawl. This single manifest tells Argo CD to watch the k8s/base/ directory in the repository, automatically sync any manifest changes to the firecrawl namespace in the cluster, prune resources deleted from Git, and self-heal any manual cluster changes back to the Git-defined state.

Purpose: This is the critical bridge between CI (Phase 1) and the cluster. Once applied, every manifest commit from the CI workflow automatically becomes a deployment without any human intervention or direct cluster access.

Output: k8s/argocd/application.yaml committed to Git, then applied to cluster by user.
</objective>

Create directory k8s/argocd/ and write k8s/argocd/application.yaml with the following exact content:

```yaml
apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: firecrawl
  namespace: argocd
  finalizers:
    - resources-finalizer.argocd.argoproj.io
spec:
  project: default
  source:
    repoURL: https://github.com/mendableai/firecrawl.git
    targetRevision: main
    path: k8s/base
  destination:
    server: https://kubernetes.default.svc
    namespace: firecrawl
  syncPolicy:
    automated:
      prune: true
      selfHeal: true
      allowEmpty: false
    syncOptions:
      - CreateNamespace=false
      - PrunePropagationPolicy=foreground
      - PruneLast=true
    retry:
      limit: 5
      backoff:
        duration: 5s
        factor: 2
        maxDuration: 3m
```

Key configuration rationale:
- **metadata.namespace: argocd** -- Application resource lives in argocd namespace where the controller runs, NOT in firecrawl namespace
- **finalizers** -- resources-finalizer ensures Argo CD cleans up managed resources if the Application is deleted
- **source.path: k8s/base** -- points to the Kustomize directory Phase 1 created; Argo CD auto-detects kustomization.yaml
- **source.targetRevision: main** -- tracks the main branch where CI commits manifest updates
- **source.repoURL** -- the firecrawl GitHub repository (same repo, monorepo approach per project decision)
- **destination.server: https://kubernetes.default.svc** -- in-cluster deployment (Argo CD runs in the same cluster)
- **destination.namespace: firecrawl** -- matches namespace in deployment manifests
- **syncPolicy.automated.prune: true** -- GITOPS-03: deletes cluster resources removed from Git
- **syncPolicy.automated.selfHeal: true** -- GITOPS-04: reverts manual kubectl changes back to Git state
- **allowEmpty: false** -- safety guard: prevents Argo CD from deleting all resources if manifests are accidentally emptied
- **CreateNamespace=false** -- namespace created in Phase 3 (FOUND-01), not by Argo CD
- **PrunePropagationPolicy=foreground** -- waits for dependent resources to be deleted before parent
- **PruneLast=true** -- prunes resources after all other sync operations complete (safer ordering)
- **retry** -- 5 retries with exponential backoff for transient failures during sync

The file is placed in k8s/argocd/ (separate from k8s/base/) to prevent Argo CD from managing its own Application resource, which would create a self-management loop.

NOTE: Argo CD auto-detects Kustomize when it finds kustomization.yaml in the source path. No explicit sourceType configuration is needed.

Human checkpoint: User applies the Application manifest to the cluster and verifies Argo CD integration.

1. Push the commit to main branch (or ensure it is on main)

2. Apply the Application manifest to the cluster:
   kubectl apply -f k8s/argocd/application.yaml

3. Verify the Application was created in Argo CD:
   kubectl get applications -n argocd firecrawl
   Expected: Shows "firecrawl" application with status

4. Check sync status (may take up to 3 minutes for first sync):
   kubectl get applications -n argocd firecrawl -o jsonpath='{.status.sync.status}'
   Expected: "Synced"

5. Check health status:
   kubectl get applications -n argocd firecrawl -o jsonpath='{.status.health.status}'
   Expected: "Healthy" or "Progressing" (resources may be progressing on first deploy)

6. Verify Argo CD dashboard shows the application:
   Open Argo CD UI (port-forward if needed: kubectl port-forward svc/argocd-server -n argocd 8080:443)
   Navigate to Applications view
   Confirm "firecrawl" application is visible with sync status and health indicators

7. Verify auto-sync works (GITOPS-02):
   The CI workflow commits to k8s/base/kustomization.yaml on push to main
   Argo CD should detect and sync within 3 minutes (default polling interval)

8. Verify prune works (GITOPS-03):
   If you temporarily add then remove a resource from k8s/base/, Argo CD should delete it from the cluster

9. Verify self-heal works (GITOPS-04):
   Run: kubectl scale deployment firecrawl-api -n firecrawl --replicas=3
   Within seconds, Argo CD should revert replicas back to 1 (Git-defined value)

NOTE: If the firecrawl namespace does not exist yet, the initial sync will show "OutOfSync" or error. This is expected -- Phase 3 creates the namespace. The Application resource itself is still valid and will sync successfully once the namespace exists.

NOTE: If the repository is private, Argo CD needs repository credentials configured. Check:
   kubectl get secrets -n argocd -l argocd.argoproj.io/secret-type=repository
If no repository secret exists for github.com/mendableai/firecrawl, add one:
   argocd repo add https://github.com/mendableai/firecrawl.git --username git --password GITHUB_PAT
